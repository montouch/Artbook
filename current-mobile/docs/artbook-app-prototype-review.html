Products, services, tickets, classes, digital assets, reviews, route checks and checkout holds.

Recent Improvement Pass

Discovery and locality

Country, city and locality now use Artbook-styled picker buttons instead of plain dropdowns. Discover and Tour can use manual scenes or GPS scene matching.

GPS behavior

The native wrapper requests Android location permission, then the app maps the phone fix to the nearest supported Artbook scene. Exact GPS should remain private unless the user chooses otherwise.

Main result: searching, filtering and location selection now stay visually aligned with themes and are more touch-friendly.

Location settings

Users can switch between phone GPS and profile/manual area. Discover and Tour areas can differ, which supports local discovery and exploring another city.

Backend requirement

Production needs server geocoding, privacy precision, spoofing controls and distance-aware ranking. Location must also affect delivery and pickup availability.

What I Would Improve Next

Release wrapper

Move from prototype WebView packaging to a proper Android project with release signing, Play Console metadata and permission rationale screens.

Backend truth

Keep all sensitive state server-owned: balances, identity, orders, delivery, Seals, reports and media rights.

Admin console

Moderation, seller verification, event legitimacy, report triage and payout holds need an operations dashboard.

Design QA

Continue testing each theme on small Android screens and add accessibility contrast checks before public launch.

Prototype truth: the APK is functional for review and product conversation, but payments, KYC, calls, delivery and uploads are still simulated until the backend and providers are connected.
